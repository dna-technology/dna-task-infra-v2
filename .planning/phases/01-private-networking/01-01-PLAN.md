  <action>
Append 4 new resource blocks to the end of `vpc.tf` (after line 84, the last `aws_route_table_association.public` resource). Follow the exact patterns from the existing file.

Add these resources in order, with section comment headers matching the existing style (`# VPC`, `# Internet Gateway`, etc.):

1. **`aws_eip.nat`** — Elastic IP for NAT Gateway
   - `domain = "vpc"` (NOT `vpc = true` — deprecated in provider 5.x)
   - Tags: `merge(module.this.tags, { Name = "${module.this.id}-nat-eip" })`

2. **`aws_nat_gateway.main`** — NAT Gateway in first public subnet
   - `allocation_id = aws_eip.nat.id`
   - `subnet_id = aws_subnet.public[0].id` (MUST be public subnet, not private)
   - `depends_on = [aws_internet_gateway.main]` (CRITICAL — NAT needs IGW attached to VPC first; Terraform cannot infer this dependency)
   - Tags: `merge(module.this.tags, { Name = "${module.this.id}-nat" })`

3. **`aws_route_table.private`** — Private route table with NAT route
   - `vpc_id = aws_vpc.main.id`
   - Inline `route` block: `cidr_block = "0.0.0.0/0"`, `nat_gateway_id = aws_nat_gateway.main.id` (follow same inline route pattern as `aws_route_table.public`)
   - Tags: `merge(module.this.tags, { Name = "${module.this.id}-private-rt" })`

4. **`aws_route_table_association.private`** — Associate all 3 private subnets
   - `count = length(aws_subnet.private)` (matches existing public pattern using `length()`)
   - `subnet_id = aws_subnet.private[count.index].id`
   - `route_table_id = aws_route_table.private.id`
   - No tags (route table associations don't support tags — same as existing public association)

After writing, run `tofu fmt vpc.tf` to ensure consistent formatting.
